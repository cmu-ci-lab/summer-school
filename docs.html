Environment Setup

The base environment is camera-agnostic: it installs everything except the vendor camera SDK, since which camera gets used is decided later. Set it up once, then bolt on the SDK for whichever camera you end up with.

# base environment (stage + processing + visualizer, no camera SDK)
conda env create -f environment.yml
conda activate iccp-oct

# or let the script do it (also handles libusb for the stage on macOS)
./setup.sh

Camera SDK — install as needed

ThorlabsStage stage.py

Wraps pylablib.devices.Thorlabs.KinesisMotor. Auto-detects the first connected Brushed Motor Controller and reads the stage type from device EEPROM (scale="stage") — so the correct encoder scale is applied automatically regardless of which compatible stage is attached.

Basic usage

from stage import ThorlabsStage

with ThorlabsStage(units="mm") as stage:
    stage.home()
    stage.move_to(10)           # absolute, 10 mm
    stage.move_by(0.5)          # relative, +0.5 mm
    print(stage.position)       # current position in mm

Unit selection

Pass units="m" or units="mm" at construction. All inputs and outputs (move_to, move_by, position) use the chosen unit. Internally pylablib always works in metres; the class multiplies/divides transparently.

AVTCamera capture_camera.py

Wraps vmbpy. On connect it scans all cameras, filters for Allied Vision by vendor name, opens the first match, and creates save_dir if it doesn't exist.

Single frame

from capture_camera import AVTCamera

with AVTCamera(exposure_us=10000, gain_db=0.0, save_dir="captures") as cam:
    image = cam.capture()          # raw uint8 or uint16, shape (H, W)
    image8 = cam.capture_8bit()    # rescaled uint8
    cam.save(image, cam.timestamped_filename())

Buffered stack acquisition

Use this pattern when stepping the stage between frames. Each call to capture_to_buffer() appends to an internal list. save_stack() stacks them to (H, W, N), saves as .npy, and clears the buffer.

for i in range(200):
    stage.move_to(i * 0.005 + 18)    # 5 µm steps from 18 mm
    cam.capture_to_buffer()

cam.save_stack(
    cam.timestamped_filename(prefix="stack", ext="npy"),
    save_previews=True,             # also writes 8-bit PNGs to captures/captured_images/
)

Call cam.clear_buffer() to discard frames without saving — useful if a run is aborted midway.

Depth-from-Focus oct.py

Translated from MATLAB (computeMeanDiff.m + getintAmplitude.m). Measures local intensity modulation per frame — the frame with highest modulation at each pixel is the in-focus frame, giving the depth map.

Algorithm

  1. DC estimate — either global mean across all frames, or a 1-D temporal box filter of width temporal_window (running average). Boundary frames are replicated rather than zero-padded to match the original MATLAB behaviour.
  2. AC component — |frame − DC| per pixel per frame.
  3. Spatial smoothing — 2-D box filter of size patch_size × patch_size applied independently to each frame slice.
  4. Depth — argmax along the frame axis → index of best-focus frame per pixel.
from oct import compute_mean_diff
import numpy as np

frames = np.load("captures/stack.npy")       # (H, W, N)

_, md_image, _ = compute_mean_diff(
    frames[:, :, ::2],          # every other frame
    patch_size=5,
    avg_type="local",
    temporal_window=20,
)

depth  = np.argmax(md_image, axis=2)   # (H, W) — best-focus frame index
maxamp = np.max(md_image,   axis=2)   # (H, W) — modulation amplitude
ℹ️
depth is 0-indexed in Python (MATLAB returns 1-indexed). For visualisation this makes no difference; if you need physical depth in mm multiply by your step size: depth_mm = depth * 0.005.

Acquisition Script oct_scan.py

Connects both devices, homes the stage, sweeps in 5 µm steps, and saves a timestamped .npy stack with 8-bit preview PNGs.

from stage import ThorlabsStage
from capture_camera import AVTCamera

stage = ThorlabsStage(units="mm")
stage.connect()

cam = AVTCamera(exposure_us=10000, gain_db=0.0, save_dir="coin_captures")
cam.connect()

stage.home()

for i in range(200):
    stage.move_to(i * 0.005 + 18)
    cam.capture_to_buffer()
    print(f"  {stage.position:.3f} mm")

cam.save_stack(cam.timestamped_filename(prefix="stack", ext="npy"), save_previews=True)

stage.release()
cam.release()

Outputs written to coin_captures/:

  • stack_YYYYMMDD_HHMMSS.npy — raw frame stack, shape (H, W, 200)
  • captured_images/stack_*_frame0000.png … — rescaled 8-bit previews

Processing Script oct_process.py

Loads a saved stack and runs the depth-from-focus pipeline.

from oct import compute_mean_diff
import numpy as np
import matplotlib.pyplot as plt
from pathlib import Path

stack_path = Path("coin_captures/stack.npy")
frames = np.load(stack_path)

_, md_image, _ = compute_mean_diff(
    frames[:, :, ::2], patch_size=5, avg_type="local", temporal_window=20
)

depth  = np.argmax(md_image, axis=2)
maxamp = np.max(md_image,   axis=2)

np.save(stack_path.with_name(stack_path.stem + "_depth.npy"),  depth)
np.save(stack_path.with_name(stack_path.stem + "_maxamp.npy"), maxamp)

Outputs (saved alongside the input stack):

  • stack_depth.npy / stack_depth.png — depth map (viridis colourmap)
  • stack_maxamp.npy / stack_maxamp.png — max amplitude (inferno colourmap)
